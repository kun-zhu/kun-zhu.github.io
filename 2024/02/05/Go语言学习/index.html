2.开发环境
在我学习和使用Java的过程中，有JDK的安装（含JVM环境等），那么Go需要安装什么环境呢？
在（https://go.dev/dl/）中下载windows的SDK，下载安装包后直接点击安装即可，现在安装包安装连环境变量都自动配置配置好了，真TM省事儿。安装过后在cmd运行`go -version，返回例如go version go1.21.6 windows/amd64`这种版本信息即代表安装成功+环境变量已经配置完成。

3.操作命令
go：这是 Go 语言的核心命令，用于管理 Go 语言的代码。它有许多子命令，如 go build（用于编译代码）、go run（用于运行代码）、go get（用于下载和安装包）等。
godoc：这是 Go 语言的文档生成工具。它可以从 Go 语言的源代码中提取注释，并生成相应的文档。这使得在编写代码的同时编写文档成为可能。
gofmt：这是 Go 语言的代码格式化工具。它可以自动调整代码的格式，使其符合 Go 语言的官方编码规范。这使得 Go 语言的代码具有很高的一致性，也使得阅读和理解他人的代码变得更加容易。

4.包
这是我犯的第一个错误，我用idea创建了一个测试类，立刻去运行了一个”hello word“做测试方法，但无法运行，因为我没有把这个文件放在一个main的包中，具体规则如下：
在 Go 语言中，包是代码的基本组织单位。每个 Go 程序都由包构成。一个包由位于单一目录下的一个或多个 .go 文件组成，目录定义包的作用域。
.go 文件：这是 Go 语言的源代码文件。每个 .go 文件都属于且只属于一个包。一个包可以由多个 .go 文件组成，这些文件都需要在同一个目录下。
在每个 .go 文件的开头，都需要声明它所属的包。
在 Go 语言中，如果你想要运行一个 .go 文件，那么这个文件必须是 main 包的一部分，并且包含一个名为 main 的函数。这是因为 Go 程序的执行始于 main 包的 main 函数。
这并不意味着所有的 .go 文件都需要在 main 包中。你的项目中可能会有许多其他的包，这些包中的 .go 文件可以被 main 包或其他包导入和使用。 对于这些非 main 包的 .go 文件，不能直接运行它们，因为它们没有 main 函数作为程序的入口点。但是可以通过编写测试来运行这些文件中的函数。也可以为每个函数编写测试，然后使用 go test 命令来运行这些测试，验证你的代码是否按照预期工作。
另外，在 Go 语言中，package main 并不意味着文件必须在名为 main 的目录中。package main 是 Go 语言中的特殊声明，它表示该文件是一个可执行程序的入口点，而不是一个库。main 包必须包含一个名为 main 的函数，这个函数是程序的入口点。
必须恰当导入需要的包，缺少了必要的包或者导入了不需要的包，程序都无法编译通过。这项严格要求避免了程序开发过程中引入未使用的包。

在 Go 语言中，首字母大写的标识符是公开的，也就是说它可以被其他包访问。首字母小写的标识符则是私有的，只能在同一个包内访问。这个规则适用于所有的标识符，包括类型（type）、函数（func）、变量（var）、常量（const）等。 例如，如果你定义了一个结构体（在 Go 中，结构体相当于其他语言中的类）：

5.Go语言不用”;“来换行吗
在Go语言中，分号主要用于在同一行内分隔多个语句。但是，如果每个语句都独占一行，那么就不需要分号。这是因为Go编译器会自动在每行结束处插入分号。

6.defer 函数中最后要执行的语句（感觉类似Java的finaly）
defer语句可以有多条，以压栈形式进入执行序列，即：后写的defer语句先执行。
defer语句和return语句同时存在，先执行return，再执行defer。

7.slice 切片 。
就是一个动态长度的数组，解决数组长度固定不方便的问题。
声明方式举例：myarray := []int{1, 2, 3} 或者 var myarr []int; myarr = make([]int, 3)或者myarr := make([]int, 3,5)
一个slice由三个部分构成：指针、长度和容量。指针指向第一个slice元素对应的底层数组元素的地址，要注意的是slice的第一个元素并不一定就是数组的第一个元素。长度对应slice中元素的数目；长度不能超过容量，容量一般是从slice的开始位置到底层数据的结尾位置。内置的len和cap函数分别返回slice的长度和容量。
个人理解：如果创建一个长度为3，容量为5的切片（未指定任何数据），一开始默认切片的数据是[0,0,0]，这是长度的含义，如果我继续向其中追加数据，则长度随之增加，如果长度要超过容量，则go会自动将容量翻倍。
截取：myarr2 := myarr[0:2] 追加： myarr2 = append(myarr2, 0)

在Go语言中，反射可以用来获取接口类型变量的动态类型信息和动态值信息。在你的代码中，你使用了reflect.TypeOf(input)来获取input的类型信息。然而，这里的input是一个值类型的Student，而不是指针类型的Student。 在Go语言中，值类型和指针类型是不同的类型，它们的方法集也是不同的。对于值类型，它的方法集只包含了接收者为值类型的方法。对于指针类型，它的方法集既包含了接收者为值类型的方法，也包含了接收者为指针类型的方法。 在你的代码中，XueXi1方法的接收者是Student，也就是说，这个方法只属于指针类型的Student，而不属于值类型的Student。因此，当你使用值类型的Student来获取方法信息时，XueXi1方法不会被包含在内。 如果你想获取XueXi1方法的信息，你需要传入一个指针类型的Student。例如，你应该使用GetMethods(&stu)而不是GetMethods(stu)。

12.go mod
这是我建立项目遇到的第一个问题，在我理解，他就是像Java的maven一样，负责管理项目的依赖的东西。
记录一些常用命令：
go mod init：在当前目录下初始化一个新的模块，创建go.mod文件。
go mod tidy：确保go.mod文件中的依赖项与项目中实际使用的依赖项匹配。它会自动添加缺失的模块，删除未使用的模块。
go mod download：下载go.mod文件中指定的所有依赖项。
go mod vendor：创建一个vendor目录，并将go.mod文件中指定的所有依赖项复制到这个目录中。
go mod verify：检查当前的依赖项是否都已经被下载，并且它们的内容是否与go.sum文件中的哈希值匹配。
go mod graph：打印模块需求图。
go mod why：解释为什么需要特定的模块或包。
go mod edit：编辑go.mod文件
